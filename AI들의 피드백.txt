1) 클로드 AI의 피드백

- Critical — 화면에 아무것도 안 뜨는 직접적 원인

1. animate() 가 한 번도 호출되지 않음
animate 함수는 정의만 되어 있고, 어디서도 시작하는 코드가 없습니다. requestAnimationFrame 루프는 최초 1회 외부에서 킥오프해야 합니다.
// main.js 맨 마지막 줄에 이게 없음
requestAnimationFrame(animate); // ← 이거 추가해야 함

2. new THREE.Camera() — 잘못된 카메라 클래스
THREE.Camera()는 추상 기반 클래스라 projection matrix가 없습니다. 풀스크린 쿼드 셰이더에는 반드시 아래로 교체해야 합니다.
// 현재 (버그)
camera = new THREE.Camera();

// 수정
camera = new THREE.OrthographicCamera(-1, 1, 1, -1, 0, 1);

- High — 기능은 켜지지만 금방 망가지는 것

3. selfieSegmentation.send() 를 매 프레임 무제한 호출
animate() 루프가 60fps로 돌면서 AI 모델에 매 프레임 send를 날립니다. MediaPipe는 비동기 처리인데 결과가 오기 전에 다음 프레임이 쌓이면서 큐가 폭발합니다.
// 수정 패턴
let segPending = false;

// animate 루프 안에서
if (isPlaying && renderVideo && !segPending) {
    segPending = true;
    selfieSegmentation.send({ image: renderVideo });
}

// onResults 콜백 안에서
selfieSegmentation.onResults(results => {
    segPending = false; // ← 결과 받으면 다음 프레임 허용
    // ... 기존 코드
});


4. renderVideo.play().then(() => renderVideo.pause()) 타이밍 문제
onloadeddata는 첫 프레임 디코딩 보장이 아닙니다. VideoTexture는 영상이 실제로 play 상태여야 텍스처가 갱신됩니다. 위 코드는 바로 pause를 때려버려서 btnPlay 전까지 오른쪽 패널이 계속 검은 화면입니다.

// renderVideo.play().then(() => renderVideo.pause()); ← 이 줄 삭제
// 그냥 play()만 하고, seeked 후 pause로 첫 프레임만 고정하거나
// 아니면 btnPlay 눌렀을 때만 play 시작하도록 두면 됨

- Minor — 나중에 고쳐도 되지만 알아야 할 것

5. initEngine() 이 initGL() 보다 먼저 실행됨
maskTex = new THREE.CanvasTexture(maskCanvas) 가 renderer 생성 전에 실행됩니다. 현재는 우연히 작동하지만, 추후 renderer 의존 코드가 initEngine에 들어가면 순서 문제가 생깁니다. initGL() → initEngine() 순서로 바꾸는 걸 권장합니다.

3) ChatGPT의 피드백
코드 전체 흐름은 꽤 잘 짜여 있어. 특히 구조(베이킹 → segmentation → shader)는 맞는 방향인데, **“오른쪽 결과가 안 뜨는 핵심 원인”**은 몇 개로 거의 확정돼.

아래는 실제로 렌더가 안 나오는 주요 원인 TOP 5야:

🚨 1. selfieSegmentation.send() 타이밍 문제 (가장 유력)

if (isPlaying && renderVideo) {
    selfieSegmentation.send({ image: renderVideo });
}

👉 문제:

isPlaying이 false면 segmentation이 아예 실행 안됨
즉, maskTex가 계속 빈 상태 → shader에서 discard됨

👉 결과:
if(mask < uPrecision) discard;
→ 전부 버려짐 = 아무것도 안 보임

✅ 해결:

if (renderVideo && renderVideo.readyState >= 2) {
    selfieSegmentation.send({ image: renderVideo });
}

👉 핵심:

Play 안 눌러도 segmentation은 돌아야 함

🚨 2. 초기 tDiffuse가 null 상태
material.uniforms.tDiffuse.value = sourceTex;

이건 btnMake 이후에만 들어감.

👉 즉:

그 전까지 fragment shader에서
texture2D(tDiffuse, uv)

→ 완전히 undefined

👉 결과:

화면 검정 or 투명

✅ 해결:
initGL 직후 임시 텍스처라도 넣어줘:

const dummy = new THREE.Texture();
material.uniforms.tDiffuse.value = dummy;
🚨 3. canvas 사이즈 0 문제 (진짜 흔한 버그)
renderer.setSize(w, h);

👉 문제:

renderArea가 아직 layout 안 잡힌 상태면
→ w, h = 0

👉 결과:

WebGL 자체가 안 보임

✅ 해결:

setTimeout(syncCameraAspect, 100);

또는

window.addEventListener('load', syncCameraAspect);
🚨 4. segmentationMask 좌표 mismatch 가능성
maskCtx.drawImage(results.segmentationMask, 0, 0);

👉 문제:

segmentationMask는 내부적으로 WebGL texture 기반이라
→ drawImage가 실패하거나 뒤집힐 수 있음

✅ 안정화 방법:

maskCtx.save();
maskCtx.scale(1, -1);
maskCtx.drawImage(results.segmentationMask, 0, -maskCanvas.height);
maskCtx.restore();
🚨 5. precision threshold 너무 높음
if(mask < uPrecision) discard;

기본값:

value="0.5"

👉 문제:

MediaPipe mask는 soft edge라서
→ 대부분 0.3~0.6 사이

👉 결과:

거의 다 discard됨

✅ 해결:

uPrecision 기본값 → 0.2~0.3
